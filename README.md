# Telegram-бот модератор (Gemini 2.5 Flash)

## Что делает
1. Смотрит на каждое текстовое сообщение в чате.
2. Если находит мат/оскорбление по словарю из `badwords.txt` — отправляет
   текст сообщения в Gemini 2.5 Flash для классификации.
3. Gemini отвечает одним из вариантов:
   - `NOTHING` — мата нет, либо мат/оскорбление направлено на конкретного
     человека → бот ничего не делает.
   - `RELIGION|причина` — оскорбление религии/веры → дальше смотрим,
     **кто** нарушил:
     - **обычный участник** → 1-е нарушение — варн с причиной,
       2-е нарушение — бан;
     - **админ или создатель чата** → бот их забанить не может
       (ограничение самого Telegram — бот не банит других админов),
       поэтому вместо бана: предупреждение прямо в чат + личное
       сообщение создателю чата с деталями (кто нарушил, причина,
       текст сообщения). Если нарушитель — сам создатель, уведомлять
       некого, только предупреждение в чат.

## Словарь запрещённых слов

Файл `badwords.txt` — по одному слову/фразе на строку, без регулярных
выражений, регистр не важен. Строки, начинающиеся с `#`, игнорируются
(можно использовать как комментарии). Добавляйте/удаляйте слова —
код трогать не нужно, бот подхватит файл при следующем запуске.

## Важно про уведомление создателя

Telegram не позволяет боту писать пользователю в личку первым — сначала
**создатель чата должен сам хотя бы раз написать боту** (например,
`/start` в личных сообщениях). Если этого не было, `send_message`
вернёт ошибку, бот залогирует предупреждение и просто не отправит ЛС
(остальная логика при этом не ломается).

## Установка

```bash
pip install -r requirements.txt
```

## Настройка

Задайте переменные окружения (или впишите прямо в bot.py):

```bash
export TELEGRAM_BOT_TOKEN="ваш_токен_от_BotFather"
export GEMINI_API_KEY="ваш_ключ_от_Google_AI_Studio"
```

Токен бота получаете у @BotFather, ключ Gemini — на https://aistudio.google.com/apikey

## Права бота в чате

Бота нужно добавить в группу **администратором** с правом
"Блокировка участников" (ban/restrict), иначе бан работать не будет —
бот просто напишет в лог предупреждение и ничего не сделает.
Дополнительных прав для чтения списка админов не требуется — это
доступно боту в любом случае.

## Запуск

```bash
python bot.py
```

## Хранение варнов

Счётчик предупреждений хранится в файле `warns.json` рядом со скриптом,
формат `{"chat_id:user_id": количество_варнов}`. Для продакшена лучше
заменить на SQLite/Redis, если чатов и пользователей много — но для
одного-двух чатов json достаточно.

## Деплой на Railway

1. Залейте эти файлы в GitHub-репозиторий (bot.py, requirements.txt,
   runtime.txt, Procfile).
2. В Railway: New Project → Deploy from GitHub repo.
3. В разделе Variables добавьте `TELEGRAM_BOT_TOKEN` и `GEMINI_API_KEY`.
4. Railway сам определит `Procfile` и запустит процесс типа `worker`
   (не `web`) — важно, чтобы Railway не ждал открытый порт, иначе
   может пометить деплой как "unhealthy". Если у вас включена
   проверка на порт — в настройках сервиса выключите Health Check
   или укажите, что порт не нужен (это polling-бот, не веб-сервер).
5. `warns.json` при рестарте контейнера на Railway **не сохранится**
   (файловая система эфемерна на бесплатном плане), если это важно —
   вынесите хранение варнов в Railway Postgres/Redis addon вместо
   локального json-файла.

## Что можно донастроить
- `badwords.txt` — словарь триггер-слов для локального предфильтра
  (чтобы не гонять через ИИ вообще все сообщения).
- Текст промпта `PROMPT_TEMPLATE` — если нужно менять критерии
  "оскорбление религии vs оскорбление человека".
- Можно добавить сброс варнов по таймеру (например, обнулять через
  30 дней), сейчас варн не сгорает сам по себе.
